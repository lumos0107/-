### 1단계 · Haversine 거리 계산

모든 거리 계산의 기반. 가장 먼저 구현하고 단위 테스트까지 완료한다.

```java
public static double haversine(double lat1, double lng1, double lat2, double lng2) {
    double R = 6371000; // 지구 반지름 (미터)
    double dLat = Math.toRadians(lat2 - lat1);
    double dLng = Math.toRadians(lng2 - lng1);
    double a = Math.sin(dLat / 2) * Math.sin(dLat / 2)
             + Math.cos(Math.toRadians(lat1)) * Math.cos(Math.toRadians(lat2))
             * Math.sin(dLng / 2) * Math.sin(dLng / 2);
    double c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
    return R * c; // 미터 단위 반환
}
```

사용 위치:
- 현재 위치 → 후보 점 거리 (distance_fit 계산)
- 후보 점 → 출발점 거리 (return_cost 계산)
- 후보 점 → 장애물/편의시설 거리 (penalty/bonus 계산)

### 3단계 · Score 함수

처음엔 ①②③만 구현해도 동작한다. 나머지는 순서대로 추가한다.

```
Score(candidate) =
    + w1 × distance_fit_score     -- ① 목표 거리와 가까울수록 높음
    - w2 × slope_penalty          -- ② 경사 클수록 낮아짐
    - w3 × obstacle_penalty       -- ③ 신호등/횡단보도 가까울수록 낮아짐
    - w4 × reuse_penalty          -- ④ 이미 지난 구간이면 낮아짐
    - w5 × return_cost            -- ⑤ 후반에 복귀 어려우면 낮아짐
    + w6 × facility_bonus         -- ⑥ 편의시설 가까울수록 높아짐
    + w7 × anchor_bonus           -- ⑦ 경유지 없을 때 반환점 방향 유도
    + w8 × waypoint_bonus         -- ⑧ 경유지 지정 시 해당 방향 유도
    - w9 × direction_penalty      -- ⑨ 급격한 꺾임일수록 낮아짐
```

**각 항목 계산식**

```java

int trafficCount = ...;

// ⑧ waypoint_bonus (경유지 있을 때만, anchor_bonus 대체)
```

### 4단계 · Greedy 메인 루프

```java
while (traveled < target - tolerance) {

    // DB 3회 조회
    List<Point> candidates  = queryCandidates(current, 50);
    List<Obstacle> obstacles = queryObstacles(current, 100);
    List<Place> places       = queryPlaces(current, 500);

    Point best = null;
    double bestScore = Double.NEGATIVE_INFINITY;

    for (Point candidate : candidates) {

        double segDist   = haversine(current, candidate);
        double predicted = traveled + segDist;
        double returnDist = haversine(candidate, start);

        // 사전 필터: 귀환 불가능한 후보 제외
        if (predicted + returnDist > target + tolerance) continue;

        double score = calcScore(candidate, predicted, returnDist, obstacles, places, ...);

        if (score > bestScore) {
            bestScore = score;
            best = candidate;
        }
    }

    if (best == null) break;

    traveled += haversine(current, best);
    visited.add(best);
    route.add(best);

    // 앵커/경유지 통과 체크
    if (haversine(best, targetPoint) < 30) isTargetPassed = true;

    current = best;
}

// 루프 검증 후 Routes API 호출
if (isLoopValid(route, start, target, tolerance)) {
    callGoogleRoutesApi(route);
}
```

### 5단계 · Google Routes API 호출

알고리즘이 선정한 점 목록을 waypoints로 전달해 실제 도로 경로를 확정한다.

```
POST https://routes.googleapis.com/directions/v2:computeRoutes
```

```json
{
  "origin": { "location": { "latLng": { "latitude": 33.123, "longitude": 126.456 } } },
  "destination": { "location": { "latLng": { "latitude": 33.123, "longitude": 126.456 } } },
  "intermediates": [
    { "location": { "latLng": { "latitude": 33.124, "longitude": 126.457 } } },
    ...
  ],
  "travelMode": "WALK"
}
```

응답에서 받은 실제 경로 좌표 → `course_points` / `course_segments`에 저장한다.
